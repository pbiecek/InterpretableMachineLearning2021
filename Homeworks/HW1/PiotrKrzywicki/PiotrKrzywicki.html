Konstrukcja i podsumowanie modelu¶

Uywamy arbitralnej architektury sieci z trzema warstwami gstymi po 256 neuronw, aktywacj relu oraz warstwami dropout z $p=0.5$.

In [8]:
from tensorflow.keras.models import Model, Sequential
from tensorflow.keras.layers import *
from tensorflow.keras.optimizers import Adam
import tensorflow
import tensorflow as tf

batchsize = 512
model = Sequential([
    Dense(256, activation='relu'),
    Dropout(0.5),
    Dense(256, activation='relu'),
    Dropout(0.5),
    Dense(256, activation='relu'),
    Dropout(0.5),
    Dense(1, activation=None),
])

model.compile(
    loss='mae',
    optimizer='adam',
    metrics=[
        tf.keras.metrics.RootMeanSquaredError(),
        tf.keras.metrics.MeanSquaredError(),
        tf.keras.metrics.MeanAbsoluteError(),
    ],
)

model.build(input_shape=(batchsize, 15))
model.summary()

Podsumowanie wyników¶

Udao si zredukowa redni bd bezwzgldny ponad dwukrotnie w stosunku do modelu zainicjowanego losowo, otrzymujc redni bd bezwzgldny na poziomie 150000.

Nie jest to mao, ale biorc pod uwag odchylenie standardowe ceny wynoszce 367118.7, widzimy, e model siczego nauczy.

Z wykresw wida, e bd na zbiorze testowym stabilizuje si po okoo 100 epokach.

Oczywicie to jest tylko demonstracja na zajciach poproszono, by w tym zadaniu uy typu modeli, ktry si najbardziej lubi, nie zwaajc zbytnio na dopasowanie do wybranych z puli dostpnych zada.

Do tego typu danych zgodnie z teraniejsz wiedz powinien zosta model lasw losowych,

najlepiej oparty o framework gradient boosting np. xgboost, bdlightgbm.
